The scope of var, let, and const

VAR:

A var variable has functional scope.That means it is present and accessible only within a function.Within the scope, it can be modified and redeclared.

LET:

A let variable has a block scope. It can be declared in a block, and it can be used only within that block.This variable is like var, which can be updated within its scope, but unlike var, which cannot be redefined within its scope.

CONST:

Variables declared with the const keyword keep their values constant. Const declarations are similar to let declarations. A const variable has a block scope.The const variable remains constant within its scope.It can't be changed or declared again.

The use cases of null and undefined

NULL:

Null is an object in JavaScript. It is the purposeful absence of value.It is one of JavaScript's primitive values.The value null is an assignment value. It can be assigned to a variable as a representation of nothing.

UNDEFINED:

Undefined indicates that a variable has been declared but no value has been assigned to it. It means the value does not exist in the compiler.It is the global object.It is a feature of ECMAScript1 (ES1).

REST(Representational State Transfer) API

An API, or application programming interface, is a set of rules that define how different applications or devices can connect to and communicate with one another. A REST API is an API that adheres to the REST, or representational state transfer, design principles. As a result, REST APIs are also referred to as RESTful APIs.
A RESTful API is an interface that allows two computer systems to securely exchange data over the internet.A RESTful API performs the same basic function as browsing the internet. When a resource is required, the client contacts the server via the API. In the server application API documentation, API developers explain how the client should use the REST API.

Some of the REST architectural principles are as follows:
  • Uniform interface
  • Layered system
  • Cacheability
  • Code on demand
RESTful APIs offer the following advantages:
  • Scalability
  • Flexibility
  • Independence
